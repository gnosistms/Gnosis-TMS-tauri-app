# Editor Render-Time Glossary Highlight Plan

## Goal

Render glossary underline markup as part of the initial editor row HTML so rows do not change height after mount when glossary marks appear.

## Current Problem

- Glossary match HTML is generated by pure highlight helpers, but it is applied to mounted row DOM in a later pass.
- Virtualized rows can therefore mount, get measured, and then change layout when glossary markup is inserted.
- That delayed layout shift is contributing to scroll jumps.

## Current Code Path

- Match generation: `src-ui/app/editor-glossary-highlighting.js`
- Cached per-row glossary highlight resolution and DOM patching: `src-ui/app/editor-glossary-flow.js`
- Static row HTML render: `src-ui/app/editor-row-render.js`
- Editor row view model construction: `src-ui/app/editor-screen-model.js`
- Visible post-render glossary sync: `src-ui/app/editor-visible-glossary-sync.js`
- Virtualized row measurement: `src-ui/app/editor-virtualization.js`

## Implementation Plan

1. Extract a render-safe glossary highlight resolver.
   - Keep the current cache key logic from `editor-glossary-flow.js`.
   - Separate pure per-row highlight computation from DOM mutation.
   - Preserve direct glossary and derived glossary support.

2. Feed precomputed glossary highlights into the render path.
   - Enrich row/section data in `editor-screen-model.js` with precomputed glossary highlight HTML for static display fields.
   - Keep this limited to non-editing display fields in the first pass.

3. Render glossary markup directly in static row HTML.
   - Update `editor-row-render.js` so `data-editor-display-text` renders glossary `<mark>` markup on first paint instead of plain escaped text.
   - Continue rendering plain text for textarea editing states.

4. Stop rewriting static display text after mount.
   - In `editor-glossary-flow.js`, remove or bypass the `displayText.innerHTML = ...` rewrite for rows that already render glossary markup.
   - Keep search highlight layer behavior intact.
   - Keep footnote handling unchanged unless it also proves necessary to move earlier.

5. Keep a narrow post-render sync path for live editing.
   - Continue using DOM updates where the text is changing live while typing in a textarea.
   - Continue supporting row-level refreshes triggered by input, AI translation completion, or derived glossary updates.
   - Treat the post-render path as fallback/interactive sync rather than the primary source of static markup.

6. Keep virtualization measuring final row content.
   - No core virtualization rewrite should be required.
   - The intended result is that a mounted virtualized row already contains its final glossary markup before measurement.

7. Preserve glossary mark interactions.
   - Verify click-to-focus and caret placement on `data-editor-glossary-mark`.
   - Verify glossary tooltip payloads still work when marks are present from first paint.

8. Reduce the deferred visible-sync path after stabilization.
   - Leave `editor-visible-glossary-sync.js` in place initially as a safety net.
   - Once stable, remove redundant static-field glossary syncing so it no longer mutates already-correct display rows.

## Test Plan

1. Unit tests
   - Static row render includes glossary `<mark>` HTML at initial render.
   - Render-time path still handles direct glossary and derived glossary matches.
   - Non-editing display fields render markup; active textareas do not.

2. Browser tests
   - Glossary marks are present immediately after first editor paint.
   - Virtualized long-scroll fixture no longer shifts when glossary underlines appear.
   - Derived glossary rows still gain the correct marks when AI translation produces a new derived glossary.
   - Tooltip and click-to-focus behavior still work on render-time marks.

## Recommended Order

1. Extract pure highlight resolver
2. Add render-time markup for static display fields
3. Skip post-mount static display rewrites
4. Add and update regression coverage
5. Remove redundant deferred static-field syncing
